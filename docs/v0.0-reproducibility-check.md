## Root Cause

`cli/cli.cpp` uses C++ float-literal syntax in `cxxopts` default value strings
(e.g. `default_value("12.0f")`, `default_value("0.999f")`, `default_value("0.9f")`).
`cxxopts` attempts to parse these strings as `float` at option-registration time
(before `parse()` is called), and its lexical cast rejects the trailing `f` suffix.

Affected lines in `cli/cli.cpp`:
- Line 139: `default_value("0.999f")` — `correspondences_flag_threshold`
- Line 141: `default_value("12.0f")` — `inlier_kappa`
- Line 154: `default_value("0.9f")` — `rigid_correspondences_flag_threshold`
- Line 156: `default_value("12.0f")` — `rigid_inlier_kappa`

The fix is trivial (strip the `f` suffixes), but `.worktrees/cli/` is read-only in
this phase — no modification allowed. This is a pre-existing bug in `origin/cli`.

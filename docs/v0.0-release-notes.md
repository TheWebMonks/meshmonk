## Reproducibility Check Outcome

**Result: BUILD_FAILED** (see `docs/v0.0-reproducibility-check.md` for full details)

The `origin/cli` binary compiled successfully (cmake + ninja both exit 0), but
crashed at startup with:

```
cxxopts::exceptions::incorrect_argument_type: Argument '12.0f' failed to parse
```

Root cause: `cli/cli.cpp` uses C++ float-literal syntax (e.g. `"12.0f"`,
`"0.999f"`) in `cxxopts` `default_value()` calls. `cxxopts` rejects the
trailing `f` suffix when parsing option defaults. This is a pre-existing bug
in `origin/cli` — not introduced by this branch.

This does **not** block v0.0. The committed golden baseline in
`tests/golden/legacy_baseline/` was produced by a prior successful run and
remains the canonical Tier 3.5 reference.

## Tag Naming

The `v0.0` tag coexists with legacy tags `v0.0.1`–`v0.0.6` from the conda/MATLAB
era. These legacy tags are archived MATLAB/conda releases and will not be
re-published. The modernization track reuses the `v0.x` prefix intentionally;
`v0.0.1`–`v0.0.6` sort above `v0.0` semver-wise but belong to a separate release
lineage.

Readers: treat `v0.0`–`v0.3` as the Python-first modernization track; treat
`v0.0.1`–`v0.0.6` as the archived pre-modernization MATLAB/conda history.

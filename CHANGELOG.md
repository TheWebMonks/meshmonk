# Changelog

All notable changes to MeshMonk are documented here.
Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

> **Legacy tag note:** Git tags `v0.0.1`–`v0.0.6` are archived MATLAB/conda
> releases from the pre-modernization era and are not part of this changelog.
> The Python-first modernization track begins at `v0.0`. See
> [ADR-001](docs/decisions/ADR-001-meshmonk-modernization.md) ("v0.0
> Clarifications") for the full tag-lineage explanation.

---

## [Unreleased]

### Added

- `TypedDict` + `Unpack` type hints on all three `register` functions
  (`RigidKwargs`, `NonrigidKwargs`, `PyramidKwargs` in `meshmonk._types`)
  — gives IDE autocomplete and pyright coverage at call sites across Python
  3.10–3.13 without raising the language-version floor ([ADR-003]).
- `NDArray[np.float32]` dtype annotations on result dataclass fields.

### Changed

- Pattern A/B validation consolidated into `_prepare_arrays`; the three
  `register` functions no longer duplicate the 18-line guard ([ADR-003]).

## [0.3.0] — 2026-04-19

First PyPI-ready release. `pip install meshmonk` works globally.

### Added

- Release workflow (`.github/workflows/release.yml`) with trusted-publisher
  OIDC — tag push triggers full cibuildwheel matrix + PyPI upload.
- cibuildwheel matrix: 12 wheels across manylinux x86_64/aarch64, macOS
  arm64, Windows x86_64; Python 3.10, 3.11, cp312-abi3 (Python 3.12+).
- Stable ABI wheels (`cp312-abi3`) via nanobind `STABLE_ABI` flag — one
  wheel covers Python 3.12 and 3.13+.
- MkDocs-material documentation site (`mkdocs.yml`) with API reference and
  migration guide.
- Cross-cutting integration tests covering rigid + nonrigid + pyramid
  pipelines end-to-end.
- `demo --download` wired to GitHub Release assets (conditional on demo-data
  redistribution rights; ships as stub if unresolved).
- `displacement_field` docstring clarification (computed from position delta,
  not extracted from the transformer internals).
- `importlib.metadata` version management — `meshmonk.__version__` is now
  read from the installed package metadata, eliminating dual maintenance.

### Changed

- cibuildwheel image updated from `manylinux2014` to `manylinux_2_28`
  (AlmaLinux 8) for both CI smoke test and release builds.
- ADR-001 D2 amended: wheel count is 12 (not 6), abi3 floor is Python 3.12
  (not 3.10), musllinux and macOS x86_64 dropped.

### Fixed

- `release.yml` artifact action versions corrected (previously referenced
  non-existent v7/v8 tags).
- OIDC permissions scoped to publish/release jobs only (was workflow-level,
  exposing tokens to build jobs running third-party code).
- `demo --download` partial-file leak on network failure — truncated cache
  file is now cleaned up before re-raising.
- `urllib.request` download wrapped with 30 s timeout and user-friendly error
  message on network failure.
- `RigidTransform` API docs corrected — removed non-existent `.rotation`,
  `.translation`, `.scale` attributes; documented actual `.matrix`, `.apply()`,
  `.compose()`, `.inverse()` surface.

## [0.2.0] — 2026-04-18

Team-ready milestone. First merge to `master`; branch rename `master → main`.

### Added

- Type stubs (`meshmonk/_meshmonk_core.pyi`) auto-generated by
  `nanobind.stubgen`; `py.typed` PEP 561 marker.
- Windows CI (`windows-latest` + MSVC 19.30) — full test suite passes on
  Windows.
- cibuildwheel dry-run CI job (one manylinux2014 wheel built and tested per
  PR).
- `MeshMonkError` typed exception class with a `.code` property
  (`RegistrationError` enum) — callers can now dispatch on error type without
  string-parsing the message. **Breaking change:** `MeshMonkError` no longer
  catches all `std::runtime_error` from the call stack; unexpected C++
  exceptions from OpenMesh, Eigen, or other dependencies now surface as
  Python `RuntimeError`.
- `_prepare_arrays()` internal helper — shared Pattern A/B dispatch and
  array-preparation logic extracted from the three register functions
  (~225 lines of duplication removed).
- `rigid_params` kwarg on `nonrigid_register()` and `pyramid_register()` for
  explicit control of the rigid pre-alignment stage.
- `std::cerr` output from internal C++ classes routed through
  `meshmonk::log()` sink, so `set_log_level("silent")` suppresses all output.
- README rewrite: Python installation, API quickstart, CLI summary, CI matrix
  table, links to migration guide and ADRs.
- Migration guide (`docs/migration-from-matlab.md`) with side-by-side MATLAB
  → Python snippets.
- Analytical tests and integration tests added.

### Changed

- **BREAKING — Pattern A vs Pattern B call signatures formalised.** The two
  supported calling patterns are now enforced with explicit errors:
  - **Pattern A** (recommended): pass mesh objects or file paths via
    `floating=` / `target=` — normals are inferred or computed automatically.
  - **Pattern B** (raw arrays): pass all six arrays explicitly
    (`floating_features`, `target_features`, `floating_faces`, `target_faces`,
    `floating_flags`, `target_flags`).
  - Mixing Pattern A and Pattern B arguments in a single call now raises
    `ValueError("Cannot mix Pattern A … with Pattern B …")`. Previously the
    behavior was undefined.
- `cli/` directory deleted (dead code — ADR-001 D4).
- `demo --download` changed from exit-0 stub to exit-1 with a clear message
  (actual download URLs deferred to v0.3).

### Fixed

- `demo --download` previously exited with code 0 on unimplemented stub.

## [0.1.0] — 2026-04-17

Foundation release. `uv pip install .` works on macOS and Linux.

### Added

- nanobind Python bindings (`_meshmonk_core`) replacing the untested pybind11
  layer. All three registration pipelines exposed: `rigid_register`,
  `nonrigid_register`, `pyramid_register`.
- `scikit-build-core` PEP 517 build backend replacing `setuptools`.
- `tl::expected`-based error returns — registration failures now surface as
  typed Python exceptions instead of printing to `stderr` and continuing with
  garbage data.
- Typed Python result dataclasses: `RigidResult` (`aligned_vertices`,
  `aligned_features`, `transform`, `iterations_run`), `NonrigidResult`,
  `PyramidResult` (with `displacement_field` and `final_inlier_weights`).
- `RigidTransform` class with `.matrix` (4×4 ndarray), `.apply()`,
  `.compose()`, `.inverse()`.
- Low-level composable primitives: `compute_correspondences`,
  `compute_inlier_weights`, `compute_rigid_transform`, `compute_normals`,
  `downsample_mesh`, `scale_shift_mesh`, `features_from_vertices`.
- typer-based Python CLI (`meshmonk rigid`, `meshmonk nonrigid`,
  `meshmonk pyramid`, `meshmonk demo`).
- GIL release in all registration functions — long-running registrations no
  longer block other Python threads.
- `meshmonk.set_log_level()` to control C++ log verbosity from Python.
- `meshmonk.MeshMonkError` exception class (bound to `std::runtime_error` at
  this version; scoped in v0.2).
- GitHub Actions CI: Ubuntu 22.04/24.04, macOS arm64, Python 3.10–3.13;
  pre-commit hooks for Python (ruff) and C++ (clang-format, cmake-format).
- Eigen 3.4.0 vendored under `vendor/` (removed FetchContent network
  download at build time — `pip install` now works offline).
- Tier 1 analytical tests (synthetic rigid recovery), Tier 2 primitive
  fixture tests (8-vertex cube), Tier 5 CLI smoke tests.

### Changed

- **BREAKING — MATLAB/C++ API removed.** The entire `extern "C"` MEX API and
  all `_mex` function variants are deleted. MATLAB callers must migrate to the
  Python API. See `docs/migration-from-matlab.md` (added in v0.2).
- **BREAKING — C++ CLI removed.** `meshmonk_cli` binary is gone; replaced by
  the `meshmonk` Python CLI entry point.
- **BREAKING — Default parameter values changed** to match the MATLAB demo
  scripts (grounded reference) rather than the former C++ header defaults:

  | Parameter | Old default | New default |
  |---|---|---|
  | `rigid.num_iterations` | 20 | **80** |
  | `inlier_kappa` | 4.0 | **12.0** |
  | `correspondences_num_neighbours` | 5 | **3** |
  | `correspondences_flag_threshold` | 0.99 | **0.9** (rigid/nonrigid) |
  | `nonrigid.num_iterations` | 60 | **200** |
  | `pyramid.num_iterations` | 60 | **90** |

- MATLAB `.m` files, `library/examples/example.cpp`, and tutorial bulk files
  deleted behind the `pre-modernization` tag (ADR-001 D3).
- Public C++ namespace reorganised: load-bearing algorithms remain under
  `namespace registration`; new public API lives under `namespace meshmonk`.
- `raw new/delete` in `RigidRegistration` and `NonrigidRegistration` replaced
  with `std::unique_ptr` (exception-safety fix, no algorithmic change).

### Fixed

- `cxxopts` float-literal `f`-suffix crash in the legacy CLI (pre-existing
  bug in `origin/cli`; fix applied during modernization before CLI deletion).

## [0.0] — 2026-04-17

Scaffolding and reality capture. No library changes; no code deletions.

### Added

- `pyproject.toml` with pytest configuration and project metadata (setuptools
  backend, later replaced by scikit-build-core in v0.1).
- `tests/` directory with scaffolding: `conftest.py`, utility helpers
  `mesh_compare` (RMSE, max vertex distance, symmetric Hausdorff) and
  `obj_to_npz` converter.
- Legacy baseline goldens (Tier 3.5): `demo/rigid_output.obj` and
  `demo/pyramid_output.obj` converted to `.npz` and ingested as
  `tests/golden/legacy_baseline/` — scientific-equivalence anchors for the
  rewrite.
- Tier 1 analytical placeholder test (`test_synthetic_rigid_recovery`) —
  intentionally failing (`xfail strict`) at this tag; first green Tier 1
  lands in v0.1.
- `NOTICE` file crediting original TheWebMonks authorship and 2026
  modernization.
- `.pre-commit-config.yaml` with Python-only hooks (ruff, ruff-format)
  scoped to modernization paths — legacy C++/MATLAB files untouched.
- `.editorconfig` (LF, UTF-8, 4-space Python, 2-space YAML).
- Design docs (`history/`) and [ADR-001] formalising the modernization
  strategy.
- Demo assets relocated to `data/`; SIMPOP meshes relocated to
  `data/simulated_faces/`.
- 8-vertex cube rigid-transform fixture cherry-picked into
  `tests/fixtures/cube_rigid/`.
- Repo transferred from `TheWebMonks/meshmonk` to `jsnyde0/meshmonk`.

### Known issues at this tag

- Tier 1 analytical test ships `xfail` — `recover_rigid()` is a stub raising
  `NotImplementedError`. Resolved in v0.1.
- `meshmonk_cli` crashes at startup due to `cxxopts` `f`-suffix bug in the
  legacy `cli/cli.cpp`. Resolved in v0.1.
- Reproducibility check outcome: `BUILD_FAILED` — the legacy CLI could not be
  rebuilt on the owner's toolchain at tag time. The committed goldens come
  from a prior successful run and remain the canonical reference.
